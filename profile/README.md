# Goblin Systems

## Core Philosophy

### Software, owned by its users

Software should belong to the people who use it.
Not rented. Not gated. Not dependent on a vendor staying alive.

You run it.
You keep it.

Ownership is the default. Longevity is a requirement.

### Sovereignty over tools and intelligence

Users choose their tools. Always.

No project dictates:

- which models you use
- which providers you depend on
- which infrastructure you run

Bring your own AI.
Bring your own cloud.

Systems are built to adapt to your choices - not the other way around.

### Local-first, private by design

Your data stays with you.

All software runs locally by default.
Nothing is sent anywhere unless you explicitly decide.

No hidden pipelines.
No silent uploads.
No implicit trust.

Privacy is not a feature - it is the baseline.

### Open source as a foundation

Every system is transparent, inspectable, and modifiable.

You can:

- read it
- change it
- fork it
- run it independently

Open source is about freedom - freedom to understand, modify, and run your own software.

### Software is distilled, not written

The role of the engineer is shifting from authoring logic to shaping behaviour.
Software emerges from models through a process of distillation - guided by prompts, data, feedback, and agent orchestration.

Distillation is the act of turning general intelligence into reliable, narrow capability.

Modern AI systems are trained on the collective output of open knowledge and open source.
What is distilled from them should return to that commons.

### Build for independence

Software should survive:

- API changes
- vendor shutdowns
- shifting ecosystems

Dependencies are optional.
Core functionality is self-contained.

What we build and run today should still work tomorrow.

### Closing

This is a shift in how software is built and who it serves.

From access to ownership.
From fixed logic to shaped intelligence.

The user is not the product.
The user is the owner.

## Goblin

Prompt Goblin and Vision Goblin are expressions of the same idea: bring your own AI, bring your own cloud. 

Prompt Goblin is a privacy-oriented desktop voice-to-text app 

Vision Goblin is a desktop image editor.

Both are pluggable with AI & Cloud providers. 

## agents-io

agents-io is a distinct product family built around AI agents - how they are installed, defined, and used across tools and environments.

## Agents and skills

Agents are still underused in many workflows, but they can be valuable where identity and intent matter. Agents define the who - identity and intent engineering. Skills define the how - enablement, capabilities, and tools.
